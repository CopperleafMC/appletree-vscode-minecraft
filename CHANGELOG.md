# Change Log

## 0.0.10

- Added persistent development-world selection for `${world}` actions.
- Formalized optional standard build, test, launch, visual comparison, profiling, evidence, and deployment workflows.
- Kept custom profile actions alongside the predefined workflow catalog.

## 0.0.9

- Made project actions profile-defined instead of renderer-specific.
- Shortened the extension details and removed the duplicate README logo.
- Kept project-specific workflows in project-owned profiles.

## 0.0.8

- Added the colored pixel-tree logo to the extension store and details view.

## 0.0.7

- Refreshed the extension and Activity Bar icons from the latest Appletree artwork.

## 0.0.6

- Updated the Activity Bar container to use the revised Appletree SVG.

## 0.0.5

- Replaced the extension icon with the revised monochrome pixel artwork.

## 0.0.4

- Centered the tree roots on the voxel platform and restored visible grass in front.

## 0.0.3

- Corrected icon depth ordering so the entire tree renders in front of the voxel platform.

## 0.0.2

- Added an original voxel sapling, apple, and grass-block extension icon.
- Replaced the generic Activity Bar glyph with a monochrome pixel-art sapling mark.

## 0.0.1

- Added a profile-driven Minecraft project model and JSON schema.
- Added project and action views, command-palette actions, and detected tasks.
- Added world selection, artifact discovery, and confirmed local mod deployment.
- Added the first project-owned profile while retaining repository scripts as workflow authority.
- Added Workspace Trust restrictions for every operation that executes project code.
- Added native VS Code Test Explorer integration for profile-defined Minecraft test lanes.
- Added read-only access to the latest client log, crash report, and screenshots.
- Added project-scoped multi-root views, tasks, tests, commands, and evidence.
- Added bundled runtime schema validation and operation-lock coordination.
- Added process-tree cancellation for external Test Explorer runs.
